Introduction to XPath

By Alice — Jan 10, 2026

XPath is a query language for selecting nodes in an XML document. It can also be used on HTML documents via the document.evaluate() API.

XPath uses a path notation — similar to URLs — for navigating through the hierarchical structure of an XML document.

— W3C XPath Specification

Common axes include: child, parent, ancestor, descendant, following-sibling, and preceding-sibling.

Advanced XPath Patterns

By Bob — Feb 14, 2026

Predicates let you filter nodes. For example:

//div[@class='card'][2]

selects the second <div> element with class card.

  • Use position() for index-based selection.
  • Use contains() for partial attribute matching.
  • Use normalize-space() to trim whitespace in text nodes.
  • Combine axes with predicates for precise targeting:
    • ancestor::table//td[last()]
    • following-sibling::li[position() <= 3]

Tips for Robust Selectors

By Carol
  1. Prefer id attributes when available — they produce the shortest, most stable paths.
  2. Avoid relying solely on positional predicates ([1], [2]) for dynamic content.
  3. Use data-* attributes as stable anchors when IDs are absent.

What is the difference between XPath 1.0 and 2.0?

XPath 2.0 adds a richer type system, more built-in functions, and support for sequences. Browsers natively support XPath 1.0 via document.evaluate().

Can I use XPath in JavaScript?

Yes — use document.evaluate(xpathExpression, contextNode, null, XPathResult.ANY_TYPE, null) in modern browsers.
